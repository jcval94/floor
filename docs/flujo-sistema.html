floor / flujo de punta a punta

Como viaja la informacion desde Yahoo Finance hasta el dashboard estatico

Este documento resume solo lo que existe hoy en el repositorio. El sistema tiene una columna vertebral clara: Yahoo -> SQLite -> features -> forecasting -> JSONL/SQLite -> snapshot -> site/data -> UI. Tambien tiene varios cortes reales en estado actual: champions ausentes, forecasting roto por un archivo truncado, dashboard stale y ramas auxiliares no conectadas a la publicacion final.

Huecos y ambiguedades reales

Puntos que un desarrollador nuevo deberia saber desde el primer dia.

HallazgoArchivo(s)Lectura tecnica

Conclusiones practicas: la columna vertebral del sistema si esta clara, pero hoy el repo no esta en verde de punta a punta. Lo mas importante para reactivar el flujo completo es reparar src/models/inference.py, corregir src/models/tasks.py, regenerar data/reports/dashboard.json, y cerrar el contrato m3 entre forecasting, persistencia y frontend.
